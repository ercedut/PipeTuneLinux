# Activation Preflight

## 1. Purpose
Activation preflight decides whether a generated profile is ready for future activation, requires manual confirmation, is blocked, or remains unknown.

## 2. Why Activation Preflight Comes Before Install
PipeTune can generate filter-chain configs, but generation is not the same as safe activation. Preflight ensures metadata and hardware risks are understood before v0.3 adds any install path.

## 3. Safety Gates
Preflight checks that the config exists, appears readable, appears to be a PipeWire filter-chain config, appears generated by PipeTune where possible, has a manifest, has preamp metadata, and does not show obvious unsafe gain.

## 4. Hardware Quirk Gates
Preflight checks machine-level HDA and microphone reliability signals. If HDA pin retask or unknown routing is detected, automatic switching is not safe.

## 5. Manual Output Confirmation
Manual output confirmation means the user must confirm the physical output target before any future activation flow. This is required for quirk-sensitive profiles and machines.

## 6. Example: Headphone Profile on Normal Machine
A headphone profile with a manifest, generated PipeTune config, preamp metadata, and no hardware quirk can be `ready` for a future v0.3 install flow.

## 7. Example: Headphone Profile on HDA Retask Machine
If HDA retask is detected, a headphone profile should return `requires_confirmation`. The user must confirm the physical output target before v0.3 installation can proceed.

## 8. Example: Laptop Speaker Profile Blocked Without Safety Metadata
A laptop speaker profile without manifest safety metadata should be blocked or remain unknown until explicit safety metadata is created and reviewed.

## 9. Privacy and Local-Only Behavior
Preflight reads local generated configs, manifests, and summary hardware status. It does not upload files. Hardware and mixer data may reveal device details, so review output before sharing.

## 10. v0.3 Install Flow
v0.3.0 uses preflight results as an install gate. It installs only user-level config, requires explicit confirmation, requires hardware quirk confirmation when needed, and refuses blocked or unknown profiles.
